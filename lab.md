Лабораторная работа 11 Организация CI/CD и автоматизированного тестирования "99%
людей принимают правила игры, если у них есть право выбора." к/ф Матрица Вам
предоставляется на выбор два задания. Настройка CI/CD с использованием GitHub
Actions (работает с репозиториями на гитхабе) Настройка CI/CD с использованием
Jenkins (работает с любыми репозиториями, предоставляет инструменты для гибкой
настройки всех CI/CD процессов, широко востребован в отрасли) Задание: Настройка
CI/CD с использованием GitHub Actions Информация для ознакомления.
https://habr.com/ru/companies/intec_balance/articles/865098/

1. Разработайте простую веб-страницу, например, форму.
2. Создайте репозиторий на GitHub и выполните первый коммит в ветку main. Также
   необходимо создать ветку dev (является основной для разработки) и fix (для
   каждой новой задачи (добавление фичи, исправление бага) создается отдельная
   ветка от dev). Их также отправить в удаленный репозиторий.
3. Напишите 3-4 автоматизированных теста пользовательского интерфейса, например,
   с использованием Selenium.
4. Настройте автоматический запуск тестов при каждом отправлении изменений в
   репозиторий (GitHub Actions). Для этого необходимо через GitHub создать в
   корне проекта папку .github/workflows/, а в ней разместить файл ci.yml. В
   этом файле указываются действия, которые будут происходить при push.
5. Убедитесь, что при внесении ошибок в код тесты завершаются неуспешно.
   Настройка CI
6. Организуйте работу с ветками. Не нужно коммитить напрямую в main. Вместо
   этого используйте pull requests.
7. Выполните следующий сценарий: – переключитесь на ветку fix. – внесите
   изменения в код (например измените текст на форме или на кнопке); – сделайте
   коммит и отправьте ветку fix в репозиторий; – зайдите на GitHub. Сделайте
   pull request из ветки fix в ветку dev и справьте ошибки (если тесты не
   прошли); – когда тесты пройдут успешно, необходимо сделать merge. Код попадет
   в ветку dev. – создайте второй pull request, но на этот раз из ветки dev в
   main. Тесты снова пробегут для финальной проверки. После их успеха выполните
   слияние. Настройка CD
8. Настройте автоматическую публикацию приложения на GitHub Pages.
9. Обеспечьте, чтобы деплой происходил только при успешном прохождении тестов в
   ветке main. Задание: Настройка CI/CD с использованием Jenkins В учебных
   материалах Jenkins https://www.jenkins.io/doc/tutorials/ выберите близкий вам
   стек и выполните пошаговое руководство.
